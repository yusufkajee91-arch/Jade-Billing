## Critical Issues (App breaks for fee earners)

### PA-001: Posting codes GET is admin-only
- **User Story:** US-009 — Fee earners select posting codes when recording time
- **Code:** `src/app/api/posting-codes/route.ts` line 18 — GET blocked for non-admin
- **Impact:** Time recording form cannot load posting code dropdown for fee earners
- **Fix:** Allow GET for all authenticated users; keep POST/PATCH/DELETE admin-only

### PA-002: Fee levels GET is admin-only
- **User Story:** US-009 — Rate auto-populated from fee earner's fee level
- **Code:** `src/app/api/fee-levels/route.ts` line 17 — GET blocked for non-admin
- **Impact:** Time recording form cannot load fee level/rate data for fee earners
- **Fix:** Allow GET for all authenticated users; keep POST/PATCH/DELETE admin-only

### PA-003: Firm settings GET is admin-only
- **User Story:** US-003 (admin configures), US-013/US-014 (invoice needs firm snapshot)
- **Code:** `src/app/api/firm-settings/route.ts` line 68 — GET blocked for non-admin
- **Impact:** Invoice creation fails for fee earners — cannot snapshot firm name, VAT, bank details
- **Fix:** Allow GET for all authenticated users; keep PUT admin-only

## Functional Issues (Features unavailable to fee earners)

### PA-004: Invoice creation blocked for fee earners
- **User Story:** US-013 — "As a **fee earner**, I want to create pro forma and tax invoices..."
- **Code:** `src/app/api/invoices/route.ts` line 65 — blocks fee_earner
- **Impact:** Only admins can create invoices, contradicting core workflow
- **Fix:** Allow fee_earner to POST invoices (for their own matters)

### PA-005: Matter notes POST is admin-only
- **User Story:** US-005 — Matter notes tab is part of matter management for fee earners
- **Code:** `src/app/api/matters/[id]/notes/route.ts` — admin-only check
- **Impact:** Fee earners cannot add notes to their own matters
- **Fix:** Allow fee_earner to create notes on matters they own or are assigned to

### PA-006: Matter attachments POST is admin-only
- **User Story:** US-005 — Matter attachments tab for fee earners
- **Code:** `src/app/api/matters/[id]/attachments/route.ts` — admin-only check
- **Impact:** Fee earners cannot upload files to their own matters
- **Fix:** Allow fee_earner to upload attachments on matters they own or are assigned to

### PA-007: Fee schedules blocked for fee earners
- **User Story:** US-033 — "As a **fee earner**, I want to access the firm's trade mark tariff..."
- **Code:** `src/app/api/fee-schedules/route.ts` line 24 — all endpoints admin-only
- **Impact:** Fee earners cannot view or use fee schedules when recording time
- **Fix:** Allow GET for all authenticated users; keep POST/PATCH/DELETE admin-only

## Missing Features

### PA-008: No collections page for fee earners
- **User Story:** US-025 — "As a **fee earner**, I want to see outstanding invoices with collection status..."
- **Code:** No `/collections` route exists. Debtors page (`/debtors`) is admin-only.
- **Impact:** Fee earners have no way to view outstanding invoices for follow-up
- **Fix:** Create `/collections` page accessible to fee earners, or grant fee earner access to debtors

## Ambiguity (Not defined in user stories)

### PA-009: `assistant` role has no user stories
- **User Story:** US-002 defines only `admin` and `fee_earner`
- **Code:** Schema includes `assistant` role; implemented as near read-only throughout
- **Decision needed:** Define what assistants should access, or remove the role

### PA-010: Bank matching allows fee earners
- **User Story:** US-020/US-021 — "As an **admin**, I want to match bank statement lines..."
- **Code:** `src/app/api/bank-matches/route.ts` — POST allows fee_earner
- **Decision needed:** Should fee earners be able to create bank matches? User stories say admin-only.

### PA-011: Trust/business entry creation by fee earners
- **User Story:** US-017 — "As an **admin**, I want to record all trust receipts..."
- **User Story:** US-018 — "As an **admin**, I want to record business account income..."
- **Code:** POST endpoints allow both admin and fee_earner
- **Decision needed:** User stories say admin-only, but fee earner access may be intentional for workflow reasons.
